BERT 医疗文本分类三种训练策略对比

任务：KUAKE-QIC 中文医疗搜索意图 11 分类
设备：RTX 4060 Laptop GPU（8GB）
数据：训练 1,368 / 验证 293 / 测试 294
设置：BERT-base-chinese，FP16，8 epochs，seed 42

核心结果

策略 Accuracy Macro-F1 训练时间 峰值显存 可训练参数
随机初始化 BERT 68.37% 45.49% 74.89 秒 1,807.7 MB 102.28M（100%）
冻结预训练 BERT 59.52% 35.32% 30.42 秒 453.2 MB 8,459（0.0083%）
预训练 BERT + LoRA 77.21% 56.94% 67.55 秒 913.0 MB 303,371（0.2957%）

训练时间仅含 Trainer.train()；峰值显存为 torch.cuda.max_memory_allocated()。类别不平衡，因此 Macro-F1 比单独 Accuracy 更能反映整体分类能力。

差异分析

随机初始化

全参数训练可拟合高频类别，但小数据不足以从零学习稳定语言表示。效果高于冻结策略，却需要最多参数、时间与显存。

冻结微调

只训练分类头，速度最快、显存最低。固定通用表示无法充分适应细粒度医疗意图，因此 Accuracy 与 Macro-F1 均最低。

LoRA

仅训练 0.30% 参数并保留预训练知识。相比随机初始化，Accuracy +8.84 个百分点，显存降低约 49.5%，综合表现最佳。

适用场景

策略 适合 主要限制
随机初始化 教学对照、验证预训练价值、拥有海量领域语料 小数据下效果与资源效率较差
冻结微调 极低显存、快速原型、最低成本基线 领域适应能力有限
LoRA 小中型领域数据、多任务 Adapter、有限 GPU 极大规模领域迁移可能仍需全量微调
结论：当前小规模中文医疗分类任务应优先采用 LoRA；冻结微调适合作为低成本基线，随机初始化主要用于实验对照。
